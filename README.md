# 狐妖小红娘·王权篇媒体触发测试

当前版本：v1.39.0。

后续 Agent 接手前请先阅读 [AGENT_HANDOFF.md](./AGENT_HANDOFF.md)。其中记录了变量契约、EJS 好感度路由、状态栏架构、部署注意事项和当前验证结果。

这是一个最小可跑测试包：玩家以 `{{user}}` 扮演王权富贵。清瞳的五档关系人设目前只保留禁用空槽位，之前代写的完整核心、双形态专项和五档内容均已移除，等待作者后续单独编写。默认开场是 `【开始】`选择页，按钮可进入演武场序章或“父亲召往主殿”；两个正式开局都内嵌 MVU 原生 `<initvar>`，按钮只切换正文与对应变量快照。剧情阶段和地点完全由 MVU 额外解析模型依据正文中已经发生的事实更新，媒体脚本不再写回或纠正剧情变量。进入“此去无归”且尚未选择后，脚本只监听玩家下一次发送，并以八次拔剑交互接管该次输入。

前期时序不是连续三天速推：首夜送信疗伤后，清瞳第二天回来与王权富贵谈及道士杀妖，当天不画画；之后她花数日绕行王权山庄四周、躲避巡查并比较远近高低与光线。再次回来织锦时，王权富贵在画到一半误判她要攻击，斩断一段蛛丝并以剑相指；回看才发现断丝织出花朵，后方则是从山外最佳角度看到的王权家全景。清瞳随后说明自己环山寻找角度的缘由，并约定去更多地方，把外界织给他看，成为他的眼睛。

Luker 测试宿主已安装 ST-Prompt-Template v1.17.4，位置为 `public/scripts/extensions/third-party/ST-Prompt-Template`。扩展的 `origin` 指向官方 GitHub 仓库，另保留 Gitee 同源镜像作为网络故障时的拉取通道。

## 目录

- `config/media-assets.json`：唯一媒体配置入口。当前 `autoStageDanmaku=false`，剧情阶段自动弹幕已关闭；拔剑联动的弹幕瀑布与 BGM 由回复事件停止，不再配置固定时长。
- `src/`：MVU 加载器、Schema、开局选择页面与控制器、状态栏、媒体控制器及无尽剑幕控制器源代码。
- `worldbook/`：默认 InitVar、主殿开场初始化模板、剧情阶段、更新规则、清瞳模块化完整人设、配角、世界底色与势力条目源文件。
- `scripts/build-components.mjs`：生成可导入 JSON。
- `scripts/deploy-to-sillytavern.mjs`：生成新角色卡并绑定世界书、脚本和正则，再部署到本机酒馆。
- `tools/local-media-server.mjs`：测试用本地静态资源服务器。
- `dist/`：构建后可直接导入 SillyTavern/酒馆助手的文件。

## 构建

在本目录运行：

```powershell
node .\scripts\build-components.mjs
node .\scripts\validate-components.mjs
```

## 本地媒体服务

保持终端窗口开启：

```powershell
node .\tools\local-media-server.mjs
```

默认地址是 `http://127.0.0.1:8123/`。当前 `dream_return` 指向项目 `音乐/呦猫UNEKO - 梦回还.mp3`；保持本地媒体服务运行即可供媒体控制器与无尽剑幕播放。

浏览器首次自动播放可能被拦截。脚本会在页面中央显示“点击启用音乐”，玩家点击一次即可授权；点击“停止音乐”也会清理尚未处理的授权提示。

## 部署新角色卡到本机酒馆

```powershell
node .\scripts\deploy-to-sillytavern.mjs "D:\json脚本地下城\主体\SillyTavern"
```

该命令会创建并安装 `狐妖小红娘·王权篇·沉浸测试.png`，同时安装同名绑定世界书。覆盖旧测试版本前会自动备份到 `backups/时间戳/`。

## 导入顺序

1. 在世界书页面导入 `dist/01-王权篇测试世界书.json`，确认 `[InitVar]王权篇变量初始化勿开` 保持禁用。
2. 在“酒馆助手 → 脚本库”依次导入 `02` 至 `08`；`08` 只负责按钮切换，不再参与 MVU 初始化时序。
3. 已经有 MVU 加载器时，不要重复启用 `02-王权篇MVU加载器`。
4. 重载脚本并新建聊天，让 MVU 走标准 InitVar 初始化流程。

## MVU 额外模型解析

本卡已经按额外模型解析的硬路由规范适配：变量更新规则使用 `[mvu_update]`，剧情、人设与阶段约束使用 `[mvu_plot]`，当前变量条目不加前缀并同时提供给剧情模型和变量模型。

在“扩展 → MVU 变量框架 → 更新方式”选择“额外模型解析”。本机 Luker 当前使用“与插头相同”、自动请求开启、依次请求失败后重试 3 次、聊天消息格式、最近 2 楼上下文；无需为本卡填写或保存第二套 API 密钥。若以后改成“自定义”模型来源，再由使用者自行填写 API 地址、密钥和模型名，角色卡不会硬编码这些敏感配置。

额外模型解析是酒馆用户级设置，不随角色卡 JSON 导入。换浏览器、换用户目录或重新部署酒馆后，需要重新确认这个开关；仅导入本卡不会擅自覆盖其他角色卡共用的 MVU 设置。

`【开始】`是 swipe 0；演武场与“父亲召往主殿”分别是 swipe 1、2。MVU 新建聊天时会读取两个正式开局各自的 `<initvar>` 并生成独立变量快照，选择页按钮只调用 swipe 切换。主殿开局初始化到 `06_败露_主殿杀令`、`最终抉择=尚未选择`、夜色初临与通往主殿的长廊；黄风城止戈已经发生，清瞳已被王权家秘密扣下，但王权富贵尚不知情。好感度仍为默认 5，不因剧情阶段自动增加。

脚本库按钮提供：测试右移弹幕、测试中央弹幕、开启弹幕瀑布、关闭弹幕瀑布、测试音乐、停止音乐、测试拔剑窗口、重置拔剑进度、重置媒体触发，以及“开始无尽剑幕试炼”。弃剑路线使用无尽剑幕；持剑救走清瞳路线暂不绑定小游戏。无尽剑幕结束后不会发送消息、触发生成或推进剧情。

## 切换到 Cloudflare R2

只修改 `config/media-assets.json`：

```json
{
  "assetBaseUrl": "https://你的公开R2域名/王权篇/",
  "tracks": {
    "dream_return": {
      "src": "bgm/dream-return.mp3"
    }
  }

```

填入新曲目后重新运行构建命令。若使用独立脚本库，只重导 `dist/04-王权篇媒体资源配置.json`；若使用本项目生成的绑定角色卡，再运行部署命令即可，控制器源码无需修改。

## 阶段触发

测试剧情共十三个阶段值，其中 `08` 按最终抉择分为尾声与独立黑暗结局：

1. `00_序章_道门兵人`
2. `01_任务_蛛网疑影`
3. `02_初遇_笼中清瞳`
4. `03_黄风_怒火迷城`
5. `04_黄风_红线家法`
6. `05_黄风_一剑开天`
7. `06_败露_主殿杀令`
8. `07_决裂_此去无归`
9. `08_尾声_万水千山`（弃剑、持剑救人后的落脚）
10. `08_结局_无心之剑`（杀掉清瞳，独立黑暗结局）
11. `09_深山_代号一二五八零`
12. `10_再起_王权在手`
13. `11_前路_龙湾之约`

黄风城至龙湾的长剧情拆为四条禁用详案，由 `45-stage-detail-controller.ejs` 根据当前阶段通过 `getwi` 动态读取。这样常驻上下文只保留阶段骨架，额外解析模型仍按正文证据逐级更新变量，剧情模型则只接收当前幕所需细节。

当前默认关闭剧情阶段自动弹幕，避免剧情尚未到达时误弹；以后需要恢复时把 `config/media-assets.json` 的 `autoStageDanmaku` 改为 `true`，再重新构建和部署。同一聊天内每个阶段最多自动触发一次，需要重测时点击“重置媒体触发”。

“此去无归”的 MVU 硬门槛只要求关系败露与父命赐剑，其他现场细节交给剧情模型自然补全。阶段达标时不会立刻遮住剧情；脚本监听玩家下一次点击发送、Enter 发送或普通快捷选项发送，取消并清空原输入，再打开拔剑过程，最后只发送玩家亲自选择的固定行动。拔剑进度以当前聊天 ID 分桶保存在媒体脚本变量中，聊天变量只保留兼容镜像；额外模型解析整表回写时不会再切断联动。旧聊天若已经存在脚本固定发送的选择消息但运行态丢失，会一次性恢复到等待后续玩家发送的安全节点，不会刷新即重放效果。测试按钮打开的是安全演示模式，不会发送消息或改变剧情。

拔剑选择消息发送后，脚本记录该玩家楼层并等待 `MESSAGE_RECEIVED`。只有更晚的、类型为 `normal`、角色为 assistant 且正文非空的回复才算确认；确认后仍不播放效果，而是继续等待更晚的非空玩家消息 `MESSAGE_SENT`。玩家这次消息正常发送，同时启动弹幕瀑布和 `dream_return` 音乐；两者持续覆盖模型生成过程，直到这条玩家消息对应的正常 assistant 回复真正落地才停止。静默生成、空回复和旧楼层不会误停，同一选择只触发一次。

拔剑选择的权威副本持久化到媒体脚本变量 `decisionByChat[当前聊天ID]`，并兼容镜像到聊天变量 `hyxhn_media_runtime.decision.selection`。字段包含 `key`、`value`、`label`、`selectedAt` 与 `choiceMessageId`。其他同卡脚本应等待全局接口 `__HYXHN_WANGQUAN_MEDIA_API__` 后调用 `getDecisionSelection()`，或监听接口公开的 `events.decisionSelected`；不要直接依赖可能被额外解析覆盖的聊天镜像。

当前救人路线链路：玩家选择“弃剑”或“持剑抱起清瞳” → LLM 完成拔剑选择回复 → 玩家下一次发送正常通过并启动持续弹幕瀑布/BGM → 对应 LLM 回复落地时停止。只有弃剑路线会在该轮回复完成后继续等待，并于玩家再发送一条正常消息时启动无尽剑幕；持剑与“杀掉清瞳”路线暂不绑定小游戏。无尽剑幕结束只关闭界面，不另发成绩或剧情消息。

弃剑路线在涂山红红打开外围缺口后不会直接跳到痊愈或抵达涂山。尾声依次承接穿过缺口、临时落脚急救、清瞳主动确认同行、王权富贵第一次亲眼看见清瞳曾画出的真实天地，以及两人共同讨论下一处落脚方向。王权富贵与东方月初的贯穿重伤、清瞳被捕时的旧伤、王权剑留在父亲一方和王权山庄追捕都会继续保留为代价。
