# 狐妖小红娘·王权篇项目 Agent 交接文档

最后更新：2026-07-16
当前角色卡版本：v1.39.0
项目目录：`hyxhn`  
角色卡名称：`狐妖小红娘·王权篇·沉浸测试`

## 1. 项目目标与范围

本项目是一张 SillyTavern 酒馆助手 MVU 角色卡测试工程，围绕《狐妖小红娘》的王权富贵与清瞳篇展开。

当前已实现：

- 玩家以 `{{user}}` 扮演王权富贵，禁止把王权富贵另写成与玩家并存的 NPC。
- 默认 first_mes 为 `【开始】`选择页；演武场和“父亲召往主殿”位于 swipe 1、2，并各自内嵌完整原生 `<initvar>`。`08-王权篇开场变量同步` 只切换 swipe，严禁通过 `replaceMvuData` 或初始化事件手动写 MVU。
- 十三阶段剧情进度，由 MVU `剧情.当前阶段` 记录；`08` 按最终抉择分为万水千山与无心之剑两个阶段值。
- 前期时序不得压成连续三天：首夜清瞳以掌心大小的半人半蜘蛛妖身替风庭云送情书，王权富贵解除符咒并在墨水盘中用药水治好她；第二天练剑时故意砍空并讨论道士为何杀妖，当天不画画。此后清瞳用数日绕行王权山庄四周，避开巡查、比较远近高低、朝向与光线。首次织锦画到一半时，王权富贵误判她要攻击，斩断一段蛛丝并以剑相指；回看才发现花朵与从山外最佳角度看到的王权家全景。
- 清瞳随后依次说明自己环山跑了好几圈才找到最好角度、听说他不能出门所以织出王权家的外貌、以后还会去更多地方把外界织给他看，并说“就让我成为你的眼睛吧”。四句是固定对白锚点，随后必须把回应权交还玩家，不得替王权富贵自动收剑、感动、答应或取名。
- 第一幅画完成后由王权富贵亲自为前期妖身取名“清瞳”；她此后每天绘制外界，他逐幅珍藏。半夜吊树夜探时，清瞳第一次稳定化为约十岁外表的人形并趴在树枝上；此后永久保持第二形态，不因伤势、妖力或好感退回，也不得成人化或色情化演绎。
- 吊树夜探后，父亲与家族才因数次留手起疑并调查，很快发现清瞳。弃剑与持剑两条救人路线都必须由王权富贵抱起清瞳走出大殿；王权霸业不在大殿内阻拦或攻击，大殿内不得战斗，山庄剑阵与追击只在殿外开始。出殿后的冲突按具体分支发展，不能再概括成父亲始终不会出手。
- 弃剑路线的固定出殿时序：王权富贵护住清瞳承受剑阵重创，身上插满剑而清瞳没有新增剑伤；东方月初以纯质阳炎炼去余下飞剑与整座剑阵；王权霸业在殿外斩出一剑，由东方月初正面承受；此后王权霸业愿意放行；涂山红红已在外围解决守卫，众人在她与东方月初帮助下转入深山隐居。持剑路线不得未经正文完成相同条件便照搬此链。
- 弃剑尾声从外围缺口继续推进：四人离庄后先临时落脚并处理三人的真实伤势；清瞳须以自己的意志和行动确认同行；王权富贵第一次亲眼看见山庄外天地；最终由王权富贵与清瞳共同开始决定下一处落脚方向。不得瞬移到涂山、单轮痊愈或由东方月初、涂山红红替两人决定归属。
- `剧情.最终抉择` 记录此去无归的唯一选择；进入该阶段后脚本先监听玩家下一次发送，拦截原输入，再提供八次拔剑与三路线替代发送。
- 剧情阶段与地点只由 MVU 额外解析模型依据正文已发生事实更新。媒体控制器不监听 `BEFORE_MESSAGE_UPDATE`，也不写回 `stat_data`；它只读取已落库阶段以决定是否监听下一次发送并打开拔剑交互。
- 拔剑选择的权威副本按聊天 ID 保存到媒体脚本变量 `decisionByChat`，聊天变量 `hyxhn_media_runtime.decision.selection` 仅作兼容镜像；其他脚本通过 `__HYXHN_WANGQUAN_MEDIA_API__.getDecisionSelection()` 或 `events.decisionSelected` 读取，避免额外解析整表回写切断联动。
- 升级前已经丢失 `decision` 的聊天会扫描固定选择消息并恢复到等待后续玩家发送的节点；只恢复媒体运行态，不重放历史效果、不修改 `stat_data`。
- 拔剑选择发送后以玩家消息楼层为锚点，先确认对应的后续完整主剧情回复落地，再等待玩家下一次发送内容；该玩家消息正常发送时启动弹幕瀑布与 BGM，并持续到它对应的正常 assistant 回复落地才停止。
- 只有“弃剑”在弹幕联动对应的 LLM 回复落地后继续等待；玩家下一次正常发送消息时启动无尽剑幕，该消息仍照常生成回复。“持剑救走清瞳”和“杀掉清瞳”暂不绑定小游戏。
- 地下城肉鸽模块已经从构建器、验证器、部署器、媒体请求事件和卡内脚本中完整移除；独立 `once-roguelite` 目录仅作为未接入的历史仓库保留，不得重新打包进角色卡。
- 无尽剑幕死亡或主动退出后只关闭界面，不发送成绩、玩家行动或续写请求。
- 清瞳好感度独立于剧情阶段更新。
- 清瞳五档人设当前均为空槽位，代写的完整核心和双形态专项已经移除；EJS 仅按好感度读取作者以后填写的对应槽位。
- 五个好感度档位各有一条禁用的空白人设槽位，唯一启用的 EJS 总控通过 `getwi` 只读取当前档位槽位，供后续手写。
- 剧情阶段变化仍可触发背景音乐；顶层剧情自动弹幕由 `config/media-assets.json` 的 `autoStageDanmaku` 控制，当前默认关闭。
- 脚本库可随时启动全屏无尽剑幕试炼：八条生命、相对拖动、无限波次；死亡或退出后不发送行动、成绩或续写请求。
- 消息楼层内显示响应式状态栏：阶段、最终抉择、时间、地点、清瞳状态、好感度和心声。
- 本地音乐地址与未来 Cloudflare R2 地址通过独立配置切换。
- 漫画／动画设定为世界观基准；一条悲剧底色常驻，八个势力条目按关键词触发。

仓库只包含本角色卡工程及音乐文件。`D:\json脚本地下城\主体\SillyTavern` 是本机测试宿主，不属于本仓库，禁止复制或提交。

## 2. 必须保持的产品约束

### 玩家身份

- `{{user}}` 始终代表王权富贵本人。
- 不硬编码酒馆当前玩家名。
- 不代写 `{{user}}` 的内心、台词和未声明行动；例外仅包括用户明确授权的拔剑回忆弹幕，以及玩家亲自点击确认的三条选择消息。

### 唯一变量结构

Schema 是变量结构的唯一事实源，当前只允许以下七项数据：

```yaml
剧情:
  当前阶段: string enum
  最终抉择: 尚未选择 | 杀掉清瞳 | 弃剑 | 持剑救走清瞳
  时间: string
  地点: string
清瞳:
  状态: string
  好感度: integer 0..100
  心声: string
```

禁止新增背包、任务、战斗、经济、玩家属性或事件流水等字段，除非用户重新确认需求。

### 十三阶段剧情与分支结局

1. `00_序章_道门兵人`
2. `01_任务_蛛网疑影`
3. `02_初遇_笼中清瞳`
4. `03_黄风_怒火迷城`
5. `04_黄风_红线家法`
6. `05_黄风_一剑开天`
7. `06_败露_主殿杀令`
8. `07_决裂_此去无归`
9. `08_尾声_万水千山`
10. `08_结局_无心之剑`
11. `09_深山_代号一二五八零`
12. `10_再起_王权在手`
13. `11_前路_龙湾之约`

每轮最多推进一个相邻阶段。阶段只代表事件进度，不代表感情进度。

### 清瞳好感度分档

- `0–19`：疏离 / 警戒
- `20–39`：试探 / 有限合作
- `40–59`：信任 / 同伴
- `60–79`：亲近 / 牵挂
- `80–100`：深情 / 相守

普通一轮最多变化 `-5..+5`，重大事件最多 `-15..+15`。好感度不得因为阶段推进自动增加。

EJS 不读取剧情阶段，也不承担剧情修正。五档槽位当前全部留空，等待作者单独编写；总控明确禁止模型仅凭档位标签自行补全、虚构或推断清瞳人设。

## 4. 世界书、脚本和正则绑定

完整卡内应包含：

- 世界书：30 条。
- 酒馆助手脚本：7 支。
- scoped 正则：6 条。

世界书关键规则：

- `[InitVar]王权篇变量初始化勿开` 必须 `enabled=false`。
- `王权篇当前变量` 必须无 `[mvu_plot]` 或 `[mvu_update]` 前缀。
- EJS 条目使用 `@@private`，允许安全声明 `const` / `let`。
- 世界书 JSON 的 `entries` 键必须等于对应字符串化 `uid`。

正则顺序：

1. `01-显示隐藏变量更新`
2. `02-对AI隐藏变量更新`
3. `03-显示王权篇状态栏`
4. `04-对AI隐藏状态栏占位符`

状态栏正则只生成 `<div data-hyxhn-statusbar-mount></div>`。常驻状态栏控制器再通过 Shadow DOM 渲染 UI。不要改回正则生成 HTML 代码块：当前 TavernHelper 环境会把那种结果直接显示成代码。

## 5. 媒体系统

当前本地配置位于 `config/media-assets.json`：

```json
{
  "assetBaseUrl": "http://127.0.0.1:8123/",
  "tracks": {
    "dream_return": {
      "src": "音乐/呦猫UNEKO - 梦回还.mp3"
    }
  }
}
```

本地测试时运行：

```powershell
node .\tools\local-media-server.mjs
```

未来迁移 Cloudflare R2 时，只修改：

- `assetBaseUrl`
- 对应曲目的 `src`

然后重新构建和部署。不要把 R2 地址直接写进 `media-controller.js`。

浏览器的自动播放策略可能阻止首次播放，媒体控制器会提供“点击启用音乐”授权按钮。

## 6. 构建、验证与部署

在项目根目录运行：

```powershell
node .\scripts\build-components.mjs
node .\scripts\validate-components.mjs
```

验证器当前检查：

- 八个 JSON 构建产物可解析且字段完整。
- 世界书 30 条、uid 对齐、InitVar 禁用、双禁递归。
- 十三个阶段枚举与媒体映射一致。
- 此去无归只在阶段命中且最终抉择尚未选择时进入监听；玩家下一次发送被捕获并取消，八次拔剑后三选一替代发送玩家消息。
- 拔剑回复确认只接受更晚的 `normal` assistant 非空回复；联动还要求回复之后存在更晚的 user 非空消息。静默生成、开场白、空错误回复和重复事件不会触发；效果没有固定时长，只在该玩家消息对应回复落地后停止。
- 清瞳好感度约束及五档 EJS 路由保留，但五档内容均为空，完整核心和双形态专项不存在。
- 状态栏使用 scoped 挂载点与 Shadow DOM。
- 媒体配置未写死本机项目路径，MP3 文件存在。

本机部署命令：

```powershell
node .\scripts\deploy-to-sillytavern.mjs "D:\json脚本地下城\主体\SillyTavern"
```

部署器会：

- 备份已有同名卡与世界书到本地 `backups/`。
- 生成完整 PNG 角色卡。
- 把世界书、脚本和正则绑定进卡。
- 将角色卡与独立世界书复制到本机 SillyTavern。

部署后必须刷新酒馆并新建聊天。旧聊天不会重新执行新版 InitVar 或替换旧开场白。

已知部署坑：当前 SillyTavern 的同名 `/api/characters/import` 流程曾保留旧卡顶层字段，只更新部分嵌入内容。开发阶段优先使用本项目部署脚本，并在酒馆角色列表确认版本号为 `1.38.0`。

Luker 兼容说明：部署器接受 `package.json.name` 为 `luker` 的同构宿主；MVU 加载器会在导入上游包前补齐 Vue/Pinia ESM 所需的编译期全局标志。该兼容路径已在 Luker 2.7.0 + 酒馆助手 4.8.18 上验证。

EJS 运行依赖：Luker 已安装 ST-Prompt-Template v1.17.4，目录为 `public/scripts/extensions/third-party/ST-Prompt-Template`。本地仓库 `origin` 指向 `https://github.com/zonde306/ST-Prompt-Template.git`，`mirror` 指向 Gitee 同源镜像；GitHub 网络恢复后优先从 `origin` 更新。

## 7. 修改变量时的同步链

任何变量字段调整必须同步修改：

1. `src/mvu-schema.js`
2. `worldbook/00-initvar.yaml`
3. `worldbook/10-update-rules.txt`
4. `worldbook/20-current-variables.txt`
5. `worldbook/50-stage-persona.ejs`（如涉及人设）
6. `src/statusbar.html`
7. `src/statusbar-controller.js`
8. 相关媒体或剧情脚本
9. 构建与验证断言

Schema 未同步时，可能出现字段被静默丢弃、状态栏读取旧路径或 LLM 生成无效 JSONPatch。

## 8. 当前验证结果

最后一次验证已确认：

- 构建、JSON 解析和项目验证全部通过。
- 完整卡实际包含 30 条世界书、7 支脚本、6 条正则。
- 无尽剑幕静态验证与五分钟难度模拟通过；Luker 真机操作手感仍需实际点击脚本按钮验收。
- 酒馆真机状态栏成功显示：`序章 · 道门兵人 / 暮色将临 / 王权山庄·演武场 / 好感 5`。
- 状态栏没有代码块外露。
- ST-Prompt-Template 真机 `/ejs` 算式与 `getwi` 世界书渲染均成功。
- 单一 EJS 总控只按好感度读取五个禁用空槽位之一；槽位等待作者后续单独填写。
- 好感度边界 `0/19/20/39/40/59/60/79/80/100` 分别正确落入五档。
- 同一好感度下改变剧情阶段不改变 EJS 人设输出。
- 当前聊天由 EJS 实际读取到 `好感度=5`，命中 `00-19` 槽位。
- 顶层弹幕此前已验证可在桌面与窄窗口显示。
- v1.6.0 真机安全演示已验证：拔剑窗口顶层显示，点击 8 次后才出现三选项，思考弹幕逐次出现，测试选项不会发送消息或改变剧情。
- 当前序章聊天不会误弹拔剑窗口；王权篇脚本日志无运行错误。
- 拔剑选项的自动发送链仍固定为 `createChatMessages(role=user)` 后接 `triggerSlash('/trigger')`；无尽剑幕已完全移除这两类调用，结束后不会自动续写。

## 9. 后续建议

优先级建议：

1. 用真实模型进行多轮 MVU 推进测试，检查 JSONPatch 的稳定性。
2. 分别测试绝情线、同伴线与恋爱线，确认 EJS 人设差异自然。
3. 把 MP3 迁移到 Cloudflare R2，再只替换媒体配置。
4. 后续再增加剧情图片与图床资源，不要把大体积 base64 图片直接塞进消息和角色卡。

接手 Agent 在改动前必须先阅读 `tavern-card-builder` skill，并以本项目已验证实现和当前 SillyTavern 运行时为准。
