# 狐妖小红娘·王权篇媒体触发测试

当前版本：v1.28.0。

后续 Agent 接手前请先阅读 [AGENT_HANDOFF.md](./AGENT_HANDOFF.md)。其中记录了变量契约、EJS 好感度路由、状态栏架构、部署注意事项和当前验证结果。

这是一个最小可跑测试包：玩家以 `{{user}}` 扮演王权富贵；EJS 人设总控只按清瞳好感度读取五档槽位。默认开场是 `【开始】`选择页，按钮可进入演武场序章或“父亲召往主殿”；两个正式开局都内嵌 MVU 原生 `<initvar>`，按钮只切换正文与对应变量快照。剧情到达“此去无归”且尚未选择时，脚本先进入监听状态；玩家读完剧情后下一次尝试发送文字或普通快捷选项时，原发送会被拦截并改由八次拔剑交互接管。若回复正文已经同时写出关系暴露、父亲把剑送到面前、明确命令杀清瞳，但模型仍把阶段写成 03，控制器会在 MVU 保存该楼层前纠正为 04。拔剑选择发出并收到对应完整主剧情回复后，脚本继续等待玩家再次发送内容，此时才联动弹幕瀑布与音乐；当前测试时长为五秒。

Luker 测试宿主已安装 ST-Prompt-Template v1.17.4，位置为 `public/scripts/extensions/third-party/ST-Prompt-Template`。扩展的 `origin` 指向官方 GitHub 仓库，另保留 Gitee 同源镜像作为网络故障时的拉取通道。

## 目录

- `config/media-assets.json`：唯一媒体配置入口。当前 `autoStageDanmaku=false`，剧情阶段自动弹幕已关闭；`postChoiceEffectDurationMs=5000` 控制拔剑回复联动测试时长。
- `src/`：MVU 加载器、Schema、开局选择页面与控制器、状态栏、媒体控制器及无尽剑幕控制器源代码。
- `worldbook/`：默认 InitVar、主殿开场初始化模板、剧情阶段、更新规则、人物、世界底色与势力条目源文件。
- `scripts/build-components.mjs`：生成可导入 JSON。
- `scripts/deploy-to-sillytavern.mjs`：生成新角色卡并绑定世界书、脚本和正则，再部署到本机酒馆。
- `tools/local-media-server.mjs`：测试用本地静态资源服务器。
- `dist/`：构建后可直接导入 SillyTavern/酒馆助手的文件。

## 构建

在本目录运行：

```powershell
node .\scripts\build-components.mjs
node .\scripts\validate-components.mjs
```

## 本地媒体服务

保持终端窗口开启：

```powershell
node .\tools\local-media-server.mjs
```

默认地址是 `http://127.0.0.1:8123/`。当前 `config/media-assets.json` 的 `tracks` 已清空，不会播放内置 BGM；媒体控制器与小游戏中的 `playTrack`、停止和淡入淡出代码全部保留。以后填入新的 `dream_return` 配置即可恢复联动音乐。

浏览器首次自动播放可能被拦截。脚本会在页面中央显示“点击启用音乐”，玩家点击一次即可授权；点击“停止音乐”也会清理尚未处理的授权提示。

## 部署新角色卡到本机酒馆

```powershell
node .\scripts\deploy-to-sillytavern.mjs "D:\json脚本地下城\主体\SillyTavern"
```

该命令会创建并安装 `狐妖小红娘·王权篇·沉浸测试.png`，同时安装同名绑定世界书。覆盖旧测试版本前会自动备份到 `backups/时间戳/`。

## 导入顺序

1. 在世界书页面导入 `dist/01-王权篇测试世界书.json`，确认 `[InitVar]王权篇变量初始化勿开` 保持禁用。
2. 在“酒馆助手 → 脚本库”依次导入 `02` 至 `08`；`08` 只负责按钮切换，不再参与 MVU 初始化时序。
3. 已经有 MVU 加载器时，不要重复启用 `02-王权篇MVU加载器`。
4. 重载脚本并新建聊天，让 MVU 走标准 InitVar 初始化流程。

`【开始】`是 swipe 0；演武场与“父亲召往主殿”分别是 swipe 1、2。MVU 新建聊天时会读取两个正式开局各自的 `<initvar>` 并生成独立变量快照，选择页按钮只调用 swipe 切换。主殿开局初始化到 `03_相知_画中山河`、`最终抉择=尚未选择`、夜色初临与通往主殿的长廊；清瞳已被王权家秘密扣下，但王权富贵尚不知情。好感度仍为默认 5，不因剧情阶段自动增加。

脚本库按钮提供：测试右移弹幕、测试中央弹幕、开启弹幕瀑布、关闭弹幕瀑布、测试音乐、停止音乐、测试拔剑窗口、重置拔剑进度、重置媒体触发，以及“开始无尽剑幕试炼”。弃剑路线使用无尽剑幕；持剑救走清瞳路线暂不绑定小游戏。无尽剑幕结束后不会发送消息、触发生成或推进剧情。

## 切换到 Cloudflare R2

只修改 `config/media-assets.json`：

```json
{
  "assetBaseUrl": "https://你的公开R2域名/王权篇/",
  "tracks": {
    "dream_return": {
      "src": "bgm/dream-return.mp3"
    }
  }
}
```

填入新曲目后重新运行构建命令。若使用独立脚本库，只重导 `dist/04-王权篇媒体资源配置.json`；若使用本项目生成的绑定角色卡，再运行部署命令即可，控制器源码无需修改。

## 阶段触发

测试剧情共六幕，第六幕按最终抉择分为两个阶段值：

1. `00_序章_道门兵人`
2. `01_任务_蛛网疑影`
3. `02_初遇_笼中清瞳`
4. `03_相知_画中山河`
5. `04_决裂_此去无归`
6. `05_尾声_万水千山`（弃剑、持剑救走清瞳）
7. `05_结局_无心之剑`（杀掉清瞳，独立黑暗结局）

当前默认关闭剧情阶段自动弹幕，避免剧情尚未到达时误弹；以后需要恢复时把 `config/media-assets.json` 的 `autoStageDanmaku` 改为 `true`，再重新构建和部署。同一聊天内每个阶段最多自动触发一次，需要重测时点击“重置媒体触发”。

“此去无归”的 MVU 硬门槛只要求关系败露与父命赐剑，其他现场细节交给剧情模型自然补全。阶段达标时不会立刻遮住剧情；脚本监听玩家下一次点击发送、Enter 发送或普通快捷选项发送，取消并清空原输入，再打开拔剑过程，最后只发送玩家亲自选择的固定行动。拔剑进度保存在聊天变量中，刷新页面不会丢失；测试按钮打开的是安全演示模式，不会发送消息或改变剧情。

拔剑选择消息发送后，脚本记录该玩家楼层并等待 `MESSAGE_RECEIVED`。只有更晚的、类型为 `normal`、角色为 assistant 且正文非空的回复才算确认；确认后仍不播放效果，而是继续等待更晚的非空玩家消息 `MESSAGE_SENT`。玩家这次消息正常发送，同时启动弹幕瀑布和 `dream_return` 音乐，经过 `postChoiceEffectDurationMs` 后自动彻底停止。同一选择只触发一次。

拔剑选择还会持久化到聊天变量 `hyxhn_media_runtime.decision.selection`，字段包含 `key`、`value`、`label`、`selectedAt` 与 `choiceMessageId`。其他同卡脚本可等待全局接口 `__HYXHN_WANGQUAN_MEDIA_API__` 后调用 `getDecisionSelection()`，或监听接口公开的 `events.decisionSelected`；旧字段 `sentChoice` 保留兼容。

当前救人路线链路：玩家选择“弃剑”或“持剑抱起清瞳” → LLM 完成拔剑选择回复 → 玩家下一次发送正常通过并触发五秒弹幕/BGM。只有弃剑路线会在该轮 LLM 回复完成后继续等待，并于玩家再发送一条正常消息时启动无尽剑幕；持剑与“杀掉清瞳”路线暂不绑定小游戏。无尽剑幕结束只关闭界面，不另发成绩或剧情消息。
