# 狐妖小红娘·王权篇媒体触发测试

当前版本：v1.68.0。

后续 Agent 接手前请先阅读 [AGENT_HANDOFF.md](./AGENT_HANDOFF.md)。其中记录了变量契约、EJS 好感度路由、状态栏架构、部署注意事项和当前验证结果。

这是一个最小可跑测试包：`{{user}}` 就是玩家在本篇中的身份。清瞳拥有跨阶段核心人设与五档关系条目：核心人设定义经历、驱动力、行为、语言和信息边界，好感档位只调整信任距离与表达强度，不代替正文互动确认关系。默认开场是 `【开始】`选择页，按钮可进入演武场序章或“父亲召往主殿”；两个正式开局都内嵌 MVU 原生 `<initvar>`，按钮只切换正文与对应变量快照。剧情大纲与详案是可偏离的参考轨道，轨道状态和最短承接约束会保存偏离结果；一剑开天后可在关系败露前主动执剑夺权或携瞳远走，12580 后则可按信任与同行选择进入经典、关系修复或独立前路。进入“此去无归”且尚未选择后，脚本只监听玩家下一次发送，并以八次拔剑交互接管该次输入。

无尽剑幕的玩家光标现已替换为压缩后的透明 WebP 双人像素角色图，资产直接编码进脚本，无需外部图片地址。角色图只改变显示，碰撞继续使用独立的小判定点；小游戏仍不播放 BGM，仅保留本地操作音效。

剑幕飞弹现以自身速度向量确定朝向，绘制为带剑刃、护手、剑柄、剑首与短尾迹的金色发光小剑；本次只更换视觉图样，飞弹数量、速度和碰撞半径均未改变。

剑身使用普通图层绘制深色外轮廓、加宽渐变刃面和中央高亮剑脊，尾迹才使用加色发光；因此密集运动和深色背景下仍能辨认完整剑刃，不会只剩护手或光线。

剑刃几何现分为两段：从护手到剑尖肩部保持等宽长方形，两条刃线互相平行；只有最前端约五分之一长度收束成三角尖头，不再把整片剑身画成长三角形。

小游戏生命耗尽时不再把结束帧误写成“试炼暂停”。剧情链启动的小游戏会锁定触发玩家楼层之后的第一条助手回复，在状态栏占位符前写入短结果标记；卡内显示正则把标记渲染为内嵌双人像素图，提示词正则则对 AI 隐藏标记与图片数据。图片不创建额外楼层、不触发生成，也不会把 Base64 塞进后续上下文。

结果楼层现改用独立的 `assets/bullethell-result.webp`：原始 1448×1086 PNG 缩放为 1200×900，并以有损 WebP 压缩至约 75 KB。楼层按最大 760px 自适应展示横版插图，玩家操控角色仍使用原有透明像素小图，两项资产互不替代。

常驻阶段大纲现已从完整剧情正文压缩为全局铁律、17 阶段导航和合法分支边界，控制在约三千字符；首夜送信与墨水盘疗伤、次日问妖与数日绕山绘景、误判斩丝、取名清瞳、长期秘密日常和王权剑历史刻痕已迁入 00 至 02 阶段动态详案。黄风城、主殿、突围、夺权、私奔、12580 与关系修复继续由原有动态详案按当前阶段加载，避免每轮重复注入尚未发生的整条剧情。

两条新增自由主线彼此独立：执剑夺权线把王权剑的个人战力、继承正统、家族剑阵、长老与道盟压力同时纳入；清瞳关系未暴露时不得凭空触发杀令，关系真实暴露后才可能回流持剑突围。携瞳远走线发生在清瞳被发现前，是两人共同流浪的侠客路线，不等同于拒绝清瞳后的独行路线；是否携带王权剑会改变追索强度，12580 身份仍可在旅途中成为后续矛盾。

携瞳远走现在可继续前往涂山：需要真实赶路与边境通报，涂山红红判断是否庇护，涂山容容安排情报、资源与外交条件，当前时间线的涂山雅雅承担警戒，翠玉灵只在确有伤情且同意出诊时介入。涂山可以提供临时庇护、合作换取立足、外交对峙或继续远行，不是无条件安全屋。雅雅、容容、翠玉灵均为关键词触发的绿灯条目，不会每轮常驻注入，也不会混用五百年后的身份与力量。

私奔线已扩展为六段可变剧情：离开笼中、山路追索、山外同路、旧网追来、涂山门前与借地立足。途中可发生人族商旅与弱小妖怪同困的救援，王权追兵只能凭正文证据逐步发现清瞳，毒娘子也不能把清瞳获救后的真心改写成任务。新增毒娘子、王权追猎队、涂山边境巡守三条关键词绿灯世界书；与原有红红、雅雅、容容、翠玉灵合计七条相关人物/群像绿灯条目，均启用但不常驻。

持剑救人线已经补全为独立固定链：{{user}}持剑抱住清瞳出殿，依次击败弟子剑阵与王权家长老；王权霸业全程不亲自出手，东方月初在长老败退后接手撤离并带二人前往深山。该线不复制弃剑重伤、霸业一剑、红红清场或毒娘子事件，完成安顿后保持 `08_尾声_万水千山` 并开放自由行动。

无心之剑补充了死亡后的持续余波：{{user}}表面平静、从细节泄露复杂反应，随后封闭情感；王权霸业受恶喙兽影响，把更精确的服从误读为稳定；风庭云通过长期远观发现他的空洞；东方月初则在确认死亡后同时表现震惊、失望、悲痛、自责与对王权家的追责，通常不会直接杀死{{user}}。

关系修复线现为“蛛丝之外”五幕：东方月初救场后的边界建立、清瞳分段交出真相、共同净化受妖毒污染的山村水源、获救村民仍驱逐妖怪的苦味余波，以及由玩家决定重新相信、继续修复或明确分别。半块糖饼、重新织画和未来道路只作为可由玩家互动触发的轻糖，不会自动完成和解。

## 目录

- `config/media-assets.json`：唯一媒体配置入口。当前 `autoStageDanmaku=false`，剧情阶段自动弹幕已关闭；拔剑联动的弹幕瀑布与 BGM 由回复事件停止，不再配置固定时长。
- `src/`：MVU 加载器、Schema、开局选择页面与控制器、状态栏、媒体控制器及无尽剑幕控制器源代码。
- `worldbook/`：默认 InitVar、主殿开场初始化模板、剧情阶段、更新规则、清瞳模块化完整人设、配角、世界底色与势力条目源文件。
- `scripts/build-components.mjs`：生成可导入 JSON。
- `scripts/deploy-to-sillytavern.mjs`：生成新角色卡并绑定世界书、脚本和正则，再部署到本机酒馆。
- `tools/local-media-server.mjs`：旧版测试工具；当前内嵌 BGM 不再需要运行。
- `dist/`：构建后可直接导入 SillyTavern/酒馆助手的文件。

## 内嵌媒体

原始 `音乐/呦猫UNEKO - 梦回还.mp3` 保持不动。卡内使用完整曲长的 `音乐/呦猫UNEKO - 梦回还·内嵌压缩版.mp3`，以 128 kbps MP3 有损压缩。构建器将压缩版编码为 `data:audio/mpeg;base64,...` 写入 `04-王权篇媒体资源配置` 脚本；弹幕瀑布直接使用脚本内数据，不需要本地媒体服务器或任何云端地址。当前压缩音频约 4.00 MB，媒体脚本约 5.33 MB，完整 PNG 卡约 15.22 MB。无尽剑幕小游戏仍不调用背景音乐，只保留倒计时、受击等本地操作音效。

## 清瞳可选形态覆写

世界书内有两条默认关闭、手动开启后每轮常驻的外形开关：

- `可选形态覆写·清瞳始终小蜘蛛娘`：清瞳始终是掌心大小、上半身少女而腰部以下为完整蜘蛛腹部与八足的小蜘蛛娘，不再转为双足人形。
- `可选形态覆写·清瞳始终纯小蜘蛛`：清瞳始终是掌心大小的完整蜘蛛妖，完全没有人形头脸、头发、躯干、手臂或双腿。

通常只开启其中一条。若误把两条都打开，`清瞳始终纯小蜘蛛` 的优先级更高并成为唯一外形基准。两条都关闭时维持原设：前期是掌心大小的半人半蜘蛛妖身，后期在既定剧情节点稳定为人形。两个条目均不带模型路由前缀，因此正文模型和 MVU 变量模型会同时收到覆写，避免正文外形与 `清瞳.状态` 相互打架。

## MVU 额外模型解析

本卡已经按额外模型解析的硬路由规范适配：变量更新规则使用 `[mvu_update]`，剧情、人设与阶段约束使用 `[mvu_plot]`，当前变量条目不加前缀并同时提供给剧情模型和变量模型。

在“扩展 → MVU 变量框架 → 更新方式”选择“额外模型解析”。本机 Luker 当前使用“与插头相同”、自动请求开启、依次请求失败后重试 3 次、聊天消息格式、最近 2 楼上下文；无需为本卡填写或保存第二套 API 密钥。若以后改成“自定义”模型来源，再由使用者自行填写 API 地址、密钥和模型名，角色卡不会硬编码这些敏感配置。

额外模型解析是酒馆用户级设置，不随角色卡 JSON 导入。换浏览器、换用户目录或重新部署酒馆后，需要重新确认这个开关；仅导入本卡不会擅自覆盖其他角色卡共用的 MVU 设置。

变量模型的有效回复必须使用 `<UpdateVariable><JSONPatch>JSON 数组</JSONPatch></UpdateVariable>` 双层容器。裸数组直接放在 `<UpdateVariable>` 下会被 MVU 判为“找到了标签但更新命令无效”；构建验证器会检查这条约束，避免后续改规则时回退。

本卡在演绎指南内置候选选项防剧透协议，不修改用户级选项预设。大纲只在后台维持因果，按钮采用“选行为，不选结果”：只能写玩家基于当前已知事实可以立即执行的一步行动，不得公开下一幕、隐藏身份、NPC 后续反应或必然结果；未点击的候选项不构成正文事实。

`【开始】`是 swipe 0；演武场与“父亲召往主殿”分别是 swipe 1、2。MVU 新建聊天时会读取两个正式开局各自的 `<initvar>` 并生成独立变量快照，选择页按钮只调用 swipe 切换。主殿开局初始化到 `06_败露_主殿杀令`、`最终抉择=尚未选择`、夜色初临与通往主殿的长廊；黄风城止戈已经发生，清瞳已被王权家秘密扣下，但{{user}}尚不知情。好感度仍为默认 5，不因剧情阶段自动增加。

脚本库按钮提供：测试右移弹幕、测试中央弹幕、开启弹幕瀑布、关闭弹幕瀑布、测试音乐、停止音乐、测试拔剑窗口、重置拔剑进度、重置媒体触发，以及“开始无尽剑幕试炼”。弃剑路线使用无尽剑幕；持剑救走清瞳路线暂不绑定小游戏。无尽剑幕结束后不会发送消息、触发生成或推进剧情。

弹幕瀑布的滚动层使用 12–17px 字号、每批固定 40 条和约 72–96ms 的批次间隔，在恢复可读字号的同时提高总体密度；中央定点弹幕保持原有字号和生成节奏。

当前默认关闭剧情阶段自动弹幕，避免剧情尚未到达时误弹；以后需要恢复时把 `config/media-assets.json` 的 `autoStageDanmaku` 改为 `true`，再重新构建和部署。同一聊天内每个阶段最多自动触发一次，需要重测时点击“重置媒体触发”。

“此去无归”的 MVU 硬门槛只要求关系败露与父命赐剑，其他现场细节交给剧情模型自然补全。阶段达标时不会立刻遮住剧情；脚本监听玩家下一次点击发送、Enter 发送或普通快捷选项发送，取消并清空原输入，再打开拔剑过程，最后只发送玩家亲自选择的固定行动。拔剑进度以当前聊天 ID 分桶保存在媒体脚本变量中，聊天变量只保留兼容镜像；额外模型解析整表回写时不会再切断联动。旧聊天若已经存在脚本固定发送的选择消息但运行态丢失，会一次性恢复到等待后续玩家发送的安全节点，不会刷新即重放效果。测试按钮打开的是安全演示模式，不会发送消息或改变剧情。

拔剑选择消息发送后，脚本记录该玩家楼层并等待 `MESSAGE_RECEIVED`。只有更晚的、类型为 `normal`、角色为 assistant 且正文非空的回复才算剧情成功；确认后继续等待更晚的非空玩家消息 `MESSAGE_SENT`。玩家这次消息正常发送时启动弹幕瀑布和 `dream_return` 音乐，并把该次生成请求标记为监听目标。非空正常回复会停止效果并推进后续链；空返回、HTTP 400/502、其他请求异常、用户中止或宿主中止也会在 `GENERATION_ENDED` / `GENERATION_STOPPED` 时停止效果，但不会冒充成功回复推进小游戏。弹幕海使用的 BGM 强制单次播放；完整曲目自然播放结束时，原生 `ended` 事件会无条件停止弹幕海并立即清理音乐状态。另有 300 秒安全超时兜底。

拔剑选择的权威副本持久化到媒体脚本变量 `decisionByChat[当前聊天ID]`，并兼容镜像到聊天变量 `hyxhn_media_runtime.decision.selection`。字段包含 `key`、`value`、`label`、`selectedAt` 与 `choiceMessageId`。其他同卡脚本应等待全局接口 `__HYXHN_WANGQUAN_MEDIA_API__` 后调用 `getDecisionSelection()`，或监听接口公开的 `events.decisionSelected`；不要直接依赖可能被额外解析覆盖的聊天镜像。

当前救人路线链路：玩家选择“弃剑”或“持剑抱起清瞳” → LLM 完成拔剑选择回复 → 玩家下一次发送正常通过并启动持续弹幕瀑布/BGM → 对应 LLM 回复落地时停止。只有弃剑路线会在该轮回复完成后继续等待，并于玩家再发送一条正常消息时启动无尽剑幕；持剑与“杀掉清瞳”路线暂不绑定小游戏。无尽剑幕结束只关闭界面，不另发成绩或剧情消息。
