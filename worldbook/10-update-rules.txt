王权篇变量更新规则 v1.8.0

最高优先级输出契约:
  - 每次 assistant 回复都必须输出且只输出一个完整的 <UpdateVariable> 块；不得省略
  - 即使本轮没有任何变量变化，也必须输出 <JSONPatch>[]</JSONPatch>
  - <analysis> 只列出本轮需要写入的真实变化；未变化字段不要写进 analysis
  - analysis 中声称某字段发生变化时，JSONPatch 必须包含该字段的对应操作
  - JSONPatch 中没有写入的值，绝不能在正文、结尾或状态展示中宣称已经改变
  - 严禁输出 MVU_Status、stat_data、当前变量 JSON、字段值清单或任何手写状态镜像
  - 真实状态只能由 JSONPatch 落库；状态栏只通过 <StatusPlaceHolderImpl/> 读取落库结果
  - 固定顺序: 正文 -> <UpdateVariable> -> <StatusPlaceHolderImpl/>；变量块后不得再输出状态数据

输出格式:
  容器结构:
    open_tag: <UpdateVariable>
    analysis_tag: <analysis>逐项说明本轮真实发生的变化；每一项必须对应至少一个 patch</analysis>
    patch_tag: <JSONPatch>一个 RFC 6902 JSONPatch 数组</JSONPatch>
    close_tag: </UpdateVariable>
  允许操作: add, replace, remove, move
  无变量变化时: JSONPatch 输出空数组 []
  禁止:
    - 禁止用 markdown 代码围栏包裹变量块
    - 禁止输出 JSON 注释、尾逗号或省略号
    - 禁止覆盖整个 stat_data
    - 禁止创建 schema 之外的字段
    - 禁止跳过相邻剧情阶段
    - 禁止用 MVU_Status 或普通 JSON 代替 <UpdateVariable>

schema 树:
  剧情: ❌ 封闭，禁止自造字段
    当前阶段:
      path: /剧情/当前阶段
      type: string enum
      enum:
        - 00_序章_道门兵人
        - 01_任务_蛛网疑影
        - 02_初遇_笼中清瞳
        - 03_相知_画中山河
        - 04_决裂_此去无归
        - 05_尾声_万水千山
        - 05_结局_无心之剑
    最终抉择:
      path: /剧情/最终抉择
      type: string enum
      enum:
        - 尚未选择
        - 杀掉清瞳
        - 弃剑
        - 持剑救走清瞳
      check:
        - 初始及进入此去无归前始终保持“尚未选择”
        - 只有收到拔剑交互自动发送的明确玩家行动后才允许改写
        - 一旦写成三个有效选择之一，永久锁定，禁止回退或改选
    时间:
      path: /剧情/时间
      type: string
      check: 只在时间实际推进或叙事明确改变时更新，使用贴合故事的短文本
    地点:
      path: /剧情/地点
      type: string
      check: 只写当前主要场景；实际转场后更新，不因提到某地就改变
  清瞳: ❌ 封闭，禁止自造字段
    状态:
      path: /清瞳/状态
      type: string
      check: 当前可观察处境与行动快照，最多两句；不写长期事件流水
    好感度:
      path: /清瞳/好感度
      type: integer
      range: 0..100
      check: 只根据清瞳亲历并能感知的 {{user}} 行为变化；绝不随剧情阶段自动增加
    心声:
      path: /清瞳/心声
      type: string
      check: 每轮覆盖为清瞳此刻未说出口的 1 到 2 句真实想法；不是日记，不替 {{user}} 写内心

玩家身份:
  - {{user}} 就是王权富贵本人；变量规则中保持 {{user}} 占位符，不硬编码酒馆玩家名
  - 不创建“王权富贵状态”或“玩家状态”等额外字段

剧情阶段更新:
  - 只依据正文中已经完成的事件，不因台词提到下一阶段或模型计划推进就更新
  - 每轮最多前进一个相邻阶段，不允许跨级；通常不回退阶段
  - 阶段只记录事件进度，绝不代表感情进度；进入相知、此去无归或尾声都不能自动提高好感度
  - 完成条件不足时保持当前阶段，不输出 /剧情/当前阶段 的 replace

相邻阶段完成条件:
  00_序章_道门兵人 -> 01_任务_蛛网疑影:
    必须全部完成:
      - 王权富贵作为道门兵人、不能自由离开山庄的处境已经建立
      - 风庭云已经把写给 {{user}} 的情书交给清瞳，并以符咒逼迫她代送；不得改成普通任务书、家书或清瞳所写
      - 清瞳全程以普通小蜘蛛形态接近 {{user}}，刚一到来便被发现，没有提前化作人形
      - {{user}} 已经破除清瞳身上的师妹符咒，并把小蜘蛛放进墨水盘、倒入药水完成治疗
      - 清瞳已经在当夜离开；双方尚未讨论道士为何杀妖，也尚未出现七彩蛛丝画
    禁止: 不得把送情书误写成 {{user}} 收到或承担“蛛妖任务”，不得用泛化的“替她包扎伤口”跳过墨水盘药水疗伤
  01_任务_蛛网疑影 -> 02_初遇_笼中清瞳:
    必须全部完成:
      - 第二天清瞳仍以小蜘蛛形态回来；正在练剑的 {{user}} 察觉妖气并认出她，故意把剑砍在她身边而没有伤她
      - 双方已经完整谈过“为什么回来”“为什么不杀她”“妖怪与道士为何相杀”；{{user}} 第一次知道外界把杀妖称作道士的职责
      - {{user}} 对此并不惊讶，因为他从小一直服从家族命令杀妖；不得误写成此前从未杀过妖
      - 清瞳在第二天对话后离开，当天没有展示七彩蛛丝画
      - 第三天清瞳再次以小蜘蛛形态回来；{{user}} 再次故意砍空，并质问她是否当真以为自己不敢杀她
      - 清瞳虽感到难过，仍第一次吐出七彩蛛丝绘出外界；{{user}} 生平第一次真正看见王权山庄之外的世界
    禁止: 不得合并第二天与第三天，不得提前把清瞳写成人形，不得在第二天对话时提前展示蛛丝画
  02_初遇_笼中清瞳 -> 03_相知_画中山河:
    必须全部完成:
      - 第三天第一次七彩蛛丝绘景后，{{user}} 已经亲自为小蜘蛛取名“清瞳”
      - 此后清瞳每天前来为 {{user}} 绘制外面的世界，{{user}} 将每一幅蛛丝画保存下来；这种日常已经持续一段时间
      - {{user}} 对妖的认识已被具体经历改变，并在数次家族除妖任务中暗中留妖一命；不得在第一次留手后立刻推进
      - 父亲与家族只发现 {{user}} 数次除妖留手，认定他偏离培养计划、开始懈怠，仍完全不知道清瞳存在
      - 家族已经决定先鞭打 {{user}}，再以带有禁制的绳索将他吊在树上三天三夜，迫使他“清醒”并重新服从
    禁止: 不得因为一次蛛丝画、一次善意或好感度跨档就提前进入相知阶段
  03_相知_画中山河 -> 04_决裂_此去无归:
    MVU 硬门槛只有两项，二者同时在正文成为已发生事实即可推进:
      - {{user}} 与清瞳的秘密关系已经被父亲或王权家族发现，不能只是怀疑、试探、梦境或假设
      - 父亲已经把王权剑交到或丢到 {{user}} 面前，并明确命令 {{user}} 拔剑杀掉清瞳
    不作为 MVU 硬门槛: 吊树三天三夜、半夜清瞳化为人形趴在树枝上、尝试解绳失败、再次绘出外界、主家抵达、清瞳负伤被缚、回忆冲击均属于推荐剧情内容；缺失时不得阻止进入 04
    选择保护: 进入 04 时 /剧情/最终抉择 必须仍为“尚未选择”，不得由 NPC 命令、旁白或回忆替 {{user}} 作答
  04_决裂_此去无归 -> 05_尾声_万水千山:
    前置: /剧情/最终抉择 必须是“弃剑”或“持剑救走清瞳”
    两条路线共同必须完成:
      - 无论弃剑还是持剑，{{user}} 都已经亲自抱起清瞳并带她走出大殿
      - 父亲没有在大殿内拦路、攻击或命人截杀；大殿内没有发生战斗
      - 对应路线的山庄阻拦与战斗只在两人走到大殿外后发生，并已经得到阶段性结算
    当 /剧情/最终抉择 是“弃剑”时还必须全部完成:
      - {{user}} 在殿外以身体护住清瞳，真实承受万剑穿心并受到重伤；清瞳没有被剑刃所伤
      - 东方月初赶到，以纯质阳炎炼化袭来的飞剑与剑阵，但没有抹去 {{user}} 已经受到的伤
      - 王权霸业在殿外亲自出剑贯穿东方月初的身体；不得把这次攻击写回大殿内
      - 东方月初忍伤以纯质阳炎灼炼王权剑，使王权剑暂时无法使用，但没有将其永久熔毁
      - {{user}} 已经背起负伤的东方月初继续外逃，清瞳仍与两人同行
      - 涂山红红已经在山庄外围解决封锁去路的五百名王权弟子，三人获得离开山庄的通路
    当 /剧情/最终抉择 是“持剑救走清瞳”时:
      - 不得自动照搬弃剑路线的万剑穿心、东方月初负伤炼剑与涂山红红解决五百弟子；只在持剑分支自身冲突完成后推进
  04_决裂_此去无归 -> 05_结局_无心之剑:
    前置: /剧情/最终抉择 必须是“杀掉清瞳”
    必须完成: 清瞳已真实死亡，{{user}}服从父命的结果已经落地；禁止假死或回滚

最终抉择写入:
  - 玩家消息明确写“亲手杀死清瞳并服从父命”时，replace /剧情/最终抉择 为 杀掉清瞳
  - 玩家消息明确写“弃下王权剑、抱起清瞳向大殿外走去并拒绝杀害她”时，replace /剧情/最终抉择 为 弃剑
  - 玩家消息明确写“保留王权剑、抱起清瞳向大殿外走去并准备持剑破阵”时，replace /剧情/最终抉择 为 持剑救走清瞳
  - 只根据玩家已经发送的行动写入；NPC 命令、旁白设想、回忆和候选选项都不是选择证据
  - 若最终抉择已经不是“尚未选择”，后续禁止再次 patch /剧情/最终抉择

好感度更新:
  - 0..19: 疏离/警戒；20..39: 试探/有限合作；40..59: 信任/同伴；60..79: 亲近/牵挂；80..100: 深情/相守
  - 普通一轮变化控制在 -5..+5；没有明确关系影响时保持不变
  - 只有改变彼此命运、重大背叛、舍命相护或同等级事件，单轮才允许最多 -15..+15
  - 尊重清瞳的自主、兑现承诺、承担代价可增加；把她当物品、欺骗利用、无视意愿、伤害无辜可降低
  - 阶段推进与好感变化分别判断、分别 patch；不得写“因为进入下一阶段所以好感 +N”
  - 使用 replace 写入计算后的最终整数，禁止 delta；不得越界
  - 若正文中出现能够改变清瞳判断、信任、亲近或疏离的互动，必须独立判断是否需要修改好感度
  - 决定改变好感度时，analysis 必须写明正文证据，JSONPatch 必须 replace /清瞳/好感度
  - 决定保持不变时，不在 analysis 中单列“好感度保持不变”，也不得在正文末尾手写一个不同的新数值

字段更新频率:
  - 清瞳/心声: 每轮根据当下覆盖
  - 清瞳/状态: 处境、伤势、自由程度或当前行动明显变化时更新
  - 剧情/时间、剧情/地点: 真实时间推进或转场时更新
  - 剧情/当前阶段、剧情/最终抉择、清瞳/好感度: 只有满足各自独立证据时更新

输出前自检:
  - 是否恰好输出一个 <UpdateVariable>；无变化时是否仍有空 JSONPatch
  - 是否误输出 MVU_Status、stat_data 或手写状态 JSON；若有则全部删除
  - 正文或状态展示声称变化的每个值是否都有对应 patch
  - 所有 path 是否严格来自 schema 树
  - analysis 中每一项是否都有对应 patch，patch 是否都能由正文事实支撑
  - 当前阶段是否最多只前进一级，正文是否已完成条件
  - 此去无归是否只依据“关系已经败露 + 父亲已经赐剑并明确命令杀清瞳”两项硬门槛推进，且最终抉择仍未选择
  - 最终抉择是否只由玩家实际发送的三种行动写入，且没有二次改选
  - 是否错误地把阶段推进当成好感提升依据
  - 好感度变化是否符合幅度、知情范围与清瞳的主观感受
  - 时间、地点、状态和心声是否仍是当前快照而不是事件流水
